# 贡献指南

感谢你对JG-RPC项目的关注！

## 项目定位

这是一个**教育性质的学习项目**，主要目的是：
1. 深入理解RPC框架的实现原理
2. 学习分布式系统的核心概念
3. 提升Java网络编程和多线程能力

## 如何贡献

虽然这是个人学习项目，但我们欢迎以下形式的贡献：

### 1. 报告Bug
如果你发现了bug，请：
- 在GitHub Issues中创建新issue
- 描述bug的重现步骤
- 提供运行环境信息（Java版本、操作系统等）

### 2. 提出建议
如果你有好的想法：
- 在GitHub Issues中创建feature request
- 说明你的想法和它带来的价值
- 如果可能，提供实现思路

### 3. 改进文档
文档永远可以更好：
- 修正拼写错误
- 补充使用示例
- 改进说明的清晰度
- 添加FAQ

### 4. 贡献代码
如果你想贡献代码：

#### 步骤：
1. Fork本项目
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 打开Pull Request

#### 代码规范：
- 遵循现有代码风格
- 添加必要的注释
- 确保代码能够编译通过
- 在PR中说明你的更改
